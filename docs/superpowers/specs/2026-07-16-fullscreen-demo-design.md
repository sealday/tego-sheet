# tego-sheet 全屏本地预览设计

日期：2026-07-16

## 目标

将现有文档式多案例 Vite demo 改为一个面向人工验收的全屏工作台。页面应在整个浏览器视口内展示单个 `TegoSheet` 实例，并提供紧凑、可折叠的测试控制区，使开发者无需修改源码即可验证核心 React API 与电子表格交互。

## 非目标

- 不改变 `tego-sheet` 的公共 API、领域行为或样式契约。
- 不加入新的运行时依赖。
- 不把 demo 变成第二套产品界面或自动化测试框架。
- 不在浏览器中持久化用户数据。

## 页面结构

页面采用 `100dvh` 的两行布局：

1. 顶部测试栏高度由内容决定，保持紧凑并允许在窄屏换行。
2. 下方工作区使用剩余全部高度，单个 `TegoSheet` 根节点宽高均为 `100%`。

测试栏可折叠。折叠后仅保留标题、当前模式、状态摘要和展开按钮，使表格获得尽可能大的可用空间。页面本身不滚动；表格内部负责滚动。

## 测试控制

测试栏提供以下功能：

- 模式：切换受控与非受控模式。切换时通过 React `key` 开启一个新的挂载周期，避免违反组件的挂载期模式不变量。
- 只读：实时切换 `readOnly`。
- 语言：在 English 与简体中文之间切换。
- 重置：恢复内置示例 workbook，并清空事件日志。
- 导入 JSON：使用文本编辑区中的 legacy workbook JSON 重新挂载表格；解析失败时显示就地错误，不破坏当前 workbook。
- 导出 JSON：通过稳定 ref 的 `getValue()` 获取当前值并写回文本编辑区。
- JSON 面板：可折叠编辑 legacy workbook JSON，避免长期占用表格空间。
- 事件日志：显示最近的 `onChange`、选择、激活工作表和错误事件；日志有固定上限，最新事件在前。

表格自身保留默认工具栏、工作表标签、编辑、格式、筛选、校验、打印入口及其他已实现交互，因此测试栏不重复实现这些功能。

## 数据流

页面维护一个当前 workbook 快照、挂载周期、模式、只读状态、语言、JSON 文本和事件日志。

- 受控模式下，`value` 来自页面状态，`onChange` 接受新值并更新页面状态。
- 非受控模式下，`defaultValue` 只在挂载时注入；导出与状态摘要通过 ref 查询当前值。
- 重置、导入和模式切换都会递增挂载周期，确保挂载期配置在明确的新生命周期中生效。
- demo 不直接访问 controller、Canvas engine 或其他非公开内部对象。

## 错误处理

- JSON 解析错误显示在 JSON 面板附近，并保留当前表格状态。
- `TegoSheet` 的 `onError` 同时写入事件日志和可见状态提示。
- 导出时若 ref 尚未就绪，按钮保持禁用或显示明确提示，不抛出未处理异常。

## 响应式与可访问性

- 测试栏在窄屏换行，工作区仍占据剩余高度。
- 所有输入都有可访问名称；切换按钮暴露当前状态。
- 错误和状态提示使用适当的 live region。
- 键盘焦点样式保持清晰，测试栏操作不会意外提交活动单元格编辑。

## 验证

- TypeScript 类型检查与 ESLint 通过。
- demo 构建通过。
- 增加或更新组件级测试，覆盖受控/非受控重挂载、只读与语言切换、JSON 导入失败保护、重置、导出和事件日志上限。
- 在本地 Vite 服务中验证页面占满视口、页面无外层滚动，并人工测试编辑、格式、撤销/重做、工作表、导入/导出和语言切换。

## 验收标准

1. 打开本地 Vite URL 后，用户只看到一个占据主要视口的 `TegoSheet` 实例。
2. 浏览器窗口缩放后，表格继续填满测试栏之外的剩余区域。
3. 用户可以从测试栏切换模式、只读和语言，并重置、导入、导出 workbook。
4. 用户可以在表格内测试现有工具栏、单元格编辑、选择、滚动和工作表操作。
5. 错误不会清空或损坏当前 workbook。
6. demo 不使用任何 legacy 公共 API 或内部 engine/controller API。
